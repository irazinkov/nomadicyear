# Nomadic Year Status Tracking Rules

## Purpose

Define how project progress is tracked and kept current during migration.

## Rules

1. Keep status tracking continuously updated in the migration plan.
2. For each phase, every task must have a `% complete` value.
3. Percent completion can be estimated when exact progress is unclear.
4. Update status after each meaningful work session, not only at phase end.
5. If a task is blocked, still update `% complete` and note it as blocked.

## Current Required Format

- Add or maintain status next to each task using this format:
  - `Task description — 0%`
  - `Task description — 45%`
  - `Task description — 100%`

## Ownership Rule

- The assistant is responsible for updating task-level `% complete` values whenever work is performed.
